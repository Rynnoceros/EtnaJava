Reponse question 1 :
Une application client lourd est un application qui contient toute la logique métier.
A contrario d'un client léger qui ne contient que la partie graphique et toute la logique métier est côté serveur.



Reponse question 2 :
Un client lours possède l'avantage d'être "autonome" et de fonctionner sur un poste sans avoir besoin d'une connexion internet.
Les inconvénients sont :
 * la place disque plus importante
 * en cas de mise à jour il faut installer les mises à jour sur tous les postes


Reponse question 3 :
Le polymorphisme permet de créer une couche d'abstraction en objet.
Par exemple je souhaite créer une application qui pourrait stocker différent animaux,
à ce moment là je créé une interface Animal qui va me définir des comportements communs 
à tous mes animaux.
Je pourrais par la suite utiliser des listes d'animaux et appeler les comportements "communs"
mais derrière chaque animal répondra de manière différente à ces appels.

exemple : IAnimal possède une méthode nourriturePrefere()
La classe Chien qui implémentera IAnimal retournera "Croquettes" alors que 
la classe Oiseau retournera "Graines".
