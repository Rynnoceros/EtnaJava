Reponse question 1 :
Un serveur d'application est un environnement de l'architecture qui fournit les éléments techniques nécéssaires à l'exécution d'applications
transactionnelles web.
Il a pour rôle de s'interfacer avec un serveur HTTP, de fournir un moteur d'exécution de traitements, il permet d'accéder à des données déjà 
existantes, en les réutilants en proposant des API permettant de s'ouvrir vers l'extérieur, il permet de gérer une architecture centraliser, 
et enfin, il est initialement prévu pour pérmettre aux équipes de développement de s'affranchir des problèmatiques techniques des architectures 
web. 

Reponse question 2 :
TomCat est un serveur Web contrairement a GlassFish qui est un serveur d'application. 
La différence est que TomCat ne se concentre que sur une seule tâche, le conteneur de servlet, donc il est plus rapide et plus léger, 
mais il ne peut héberger que des applications web.
Quant à GlassFish, il permet: le déploiement EAR, le Pool de connexion, la Gestion de transactions distribuées, le traitement JMS, 
le clustring, le load balancing, et il peut héberger des applications d'entreprise EAR qui utilisent des ressources JMS
